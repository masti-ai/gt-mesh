### Task 5: Commit Message Validator Hook

**Files:**
- Create: `villa_ai_planogram/mayor/rig/.githooks/commit-msg`
- Create: `villa_alc_ai/mayor/rig/.githooks/commit-msg`

**Step 1: Write the commit-msg hook for villa_ai_planogram**

```bash
#!/usr/bin/env bash
# Commit message validator — enforces: <type>(<bead-id>): <description>
# Types: feat, fix, refactor, chore, test, docs
# Bead ID: project prefix + alphanumeric (e.g., vap-123, vaa-abc, hq-xyz)

MSG_FILE="$1"
MSG=$(head -1 "$MSG_FILE")

# Allow merge commits
if echo "$MSG" | grep -qE "^Merge "; then
    exit 0
fi

# Allow revert commits
if echo "$MSG" | grep -qE "^Revert "; then
    exit 0
fi

# Validate format: type(bead-id): description
if ! echo "$MSG" | grep -qE "^(feat|fix|refactor|chore|test|docs)\([a-z]+-[a-z0-9]+\): .+"; then
    echo ""
    echo "ERROR: Invalid commit message format."
    echo ""
    echo "Required: <type>(<bead-id>): <description>"
    echo "Types:    feat, fix, refactor, chore, test, docs"
    echo "Example:  feat(vap-123): add JWT auth to photo upload endpoint"
    echo ""
    echo "Your message: $MSG"
    echo ""
    exit 1
fi
```

**Step 2: Make it executable**

```bash

```

```markdown
# Branch Naming Convention

## Format

`<polecat-name>-<type>-<slug>`

## Components

| Component | Source | Example |
|-----------|--------|---------|
| polecat-name | Polecat identity | `aalu-bomb` |
| type | Bead type mapping | `feat` |
| slug | Bead title, kebab-case, max 5 words | `add-auth-endpoint` |

## Type Mapping

| Bead Type | Branch Type |
|-----------|-------------|
| feature | feat |
| bug | fix |
| task (refactor) | refactor |
| task (other) | chore |
| test | test |
| docs | docs |

## Examples

- `aalu-bomb-feat-add-auth-endpoint`
- `anar-fix-upload-crash`
- `mayor-chore-cleanup-stale-branches`

## Rules

1. All lowercase, kebab-case only
2. No slashes (breaks some CI tools)
3. Max total length: 63 characters (git branch limit for some remotes)
4. Slug derived from bead title — first 5 meaningful words
```

**Create GitHub PR (if GitHub remote exists):**

Before submitting to the internal merge queue, create a GitHub PR for human visibility.

1. Check if a GitHub remote exists:
\```bash

\```

2. If GitHub remote exists, create the PR:
\```bash
if [ "$HAS_GH" = "true" ]; then
  # Get bead info for PR body
  BEAD_TITLE=$(bd show {{issue}} --json | jq -r '.[0].title')
  BEAD_DESC=$(bd show {{issue}} --json | jq -r '.[0].description // "No description"')
  DIFF_STAT=$(git diff --stat origin/{{base_branch}}...HEAD)
  FILES_CHANGED=$(git diff --stat origin/{{base_branch}}...HEAD | tail -1)

  # Push branch first
  git push origin HEAD

  # Create PR with template auto-fill
  gh pr create \
    --title "[{{issue}}] <type>: ${BEAD_TITLE}" \
    --body "## Summary
${BEAD_DESC}

## Bead
\`{{issue}}\` — ${BEAD_TITLE}

## Changes
\`\`\`
${DIFF_STAT}
\`\`\`

${FILES_CHANGED}

## Testing
- [ ] Build passes
- [ ] Targeted tests pass
- [ ] Self-reviewed for security/style

---
🤖 Generated by polecat \`$(hostname)\` | Bead: \`{{issue}}\` | Rig: auto-detected" \
    --base {{base_branch}}
fi
\```

If PR creation fails (network, permissions), log the error and continue with `gt done`.
The internal MR bead is the source of truth; the GitHub PR is a human-readable mirror.

**Step 8: STALE BRANCH DETECTION**

Check for remote branches older than 7 days with no associated open bead:

\```bash
# List remote branches with last commit date
git for-each-ref --sort=-committerdate --format='%(committerdate:short) %(refname:short)' refs/remotes/origin/ | while read DATE BRANCH; do
  # Skip main, HEAD, integration branches
  echo "$BRANCH" | grep -qE '(^origin/main$|^origin/HEAD$|^origin/integration/)' && continue

  # Check age
  DAYS_OLD=$(( ($(date +%s) - $(date -d "$DATE" +%s)) / 86400 ))
  if [ "$DAYS_OLD" -gt 7 ]; then
    echo "STALE: $BRANCH ($DAYS_OLD days old)"
  fi
done
\```

For each stale branch found:
1. Check if there's an open MR bead for it: `bd list --type=merge-request --status=open | grep <branch>`
2. If no open MR: flag for cleanup
3. Collect stale branches and report to Deacon:
\```bash
gt mail send deacon/ -s "STALE_BRANCHES: <count> branches older than 7d" \
  -m "Stale branches with no open MR:
<list of branches with ages>

Recommend: gt polecat gc to clean up."
\```
