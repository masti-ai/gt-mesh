# GT Workflow Templates & Conventions Design

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Establish deterministic, template-driven workflow for how beads become GitHub issues, branches, commits, PRs, and merges — giving humans a clean GitHub dashboard while agents keep detailed tracking in beads.

**Architecture:** Two-layer model. Beads remain the source of truth for agent coordination. GitHub becomes the human-readable mirror — issues for project tracking, PRs for code review and audit trail.

---

## Two-Layer Model

### Layer 1 — Beads (Agent Layer)
- Granular, machine-readable, tracks every micro-task
- Where polecats, witnesses, and refineries coordinate
- No changes to how this works today

### Layer 2 — GitHub (Human Layer)
- High-level, visual, human-readable
- GitHub issues = feature/epic tracking (1 issue groups multiple beads)
- GitHub PRs = rich work summaries with screenshots, before/after context, linked beads

### How They Connect
- Mayor creates a GitHub issue when planning a feature → links beads inside it
- Polecat creates a GitHub PR at `gt done` → PR body references bead ID and parent GitHub issue
- Refinery merges through the GitHub PR (not raw `git merge`) → review diffs, CI history, merge audit

## Naming Conventions

### Branch Naming
**Format:** `<polecat-name>-<type>-<slug>`

| Component | Source | Example |
|-----------|--------|---------|
| polecat-name | Polecat identity | `aalu-bomb` |
| type | `feat`, `fix`, `refactor`, `chore`, `test`, `docs` | `feat` |
| slug | kebab-case from bead title, max 5 words | `add-auth-endpoint` |

**Examples:**
- `aalu-bomb-feat-add-auth-endpoint`
- `anar-fix-upload-crash`
- `mayor-chore-cleanup-stale-branches`

### Commit Messages
**Format:** `<type>(<bead-id>): <description>`

Conventional Commits with bead ID as scope. Enforced by pre-commit hook.

**Examples:**
- `feat(vap-123): add JWT auth to photo upload endpoint`
- `fix(vap-456): handle null shelf_id in planogram query`
- `test(vap-123): add unit tests for auth middleware`
- Multi-bead work: use primary bead ID, mention others in commit body

### PR Title
**Format:** `[<bead-id>] <type>: <human-readable title>`

**Examples:**
- `[vap-123] feat: add auth endpoint for mobile app`
- `[vaa-456] fix: alcohol concierge session crash on empty cart`

### GitHub Issue Title
**Format:** `[<rig>] <feature description>`

**Examples:**
- `[planogram] Photo upload and ML processing pipeline`
- `[alc-ai] Alcohol concierge checkout flow`

## End-to-End Flow

```
1. MAYOR plans work
   ├─ Creates GitHub issue (human tracker) with template
   ├─ Creates beads (agent tasks) linked to the issue
   └─ Links beads to GitHub issue in the beads table

2. MAYOR slings bead to polecat
   ├─ gt sling vap-123 villa_ai_planogram --create
   └─ Polecat spawns, gets hooked with vap-123

3. POLECAT works (existing 7-step formula, unchanged)
   ├─ Branch: aalu-bomb-feat-add-auth-endpoint
   ├─ Commits: feat(vap-123): add JWT auth
   └─ Runs gt done →
       ├─ Pushes branch
       ├─ Creates GitHub PR (using template, auto-fills from bead)
       ├─ Creates MR bead (links to PR URL)
       ├─ Updates GitHub issue bead table (status → in_review, PR → link)
       └─ Session exits

4. REFINERY picks up MR
   ├─ Reviews GitHub PR diff
   ├─ Runs build/tests/lint
   ├─ If green: merges PR via `gh pr merge --squash`
   ├─ If red: spawns fix polecat, comments on PR with failure details
   └─ After merge:
       ├─ Source branch auto-deleted by GitHub
       ├─ Updates GitHub issue bead table (status → done)
       ├─ Closes bead
       └─ Sends MERGED mail

5. CLEANUP (automatic)
   ├─ Merged branches auto-deleted by GitHub
   ├─ Polecat sandbox preserved for reuse
   ├─ GitHub issue auto-closes when all beads → done
   └─ Convoy closes
```

## What Changes in GT

### Modified: `gt done` command
- After pushing branch, also creates GitHub PR using template
- Populates PR body from bead metadata (title, description, bead ID)
- Detects frontend changes and flags screenshots as required
- Links PR URL back to the MR bead

### Modified: Refinery merge step
- Uses `gh pr merge --squash` instead of raw `git merge`
- Comments on PR with build/test results
- Updates GitHub issue bead table after merge

### New: GitHub issue creation in Mayor workflow
- When Mayor plans a feature, creates GitHub issue with template
- Links beads to the issue
- Beads table in issue body auto-updated as work progresses

### New: Pre-commit hook
- Validates commit message format: `<type>(<bead-id>): <description>`
- Rejects commits without bead ID scope

### New: Branch naming in polecat formula
- Branch created as `<polecat-name>-<type>-<slug>` instead of `polecat/<name>`
- Type derived from bead type (feature→feat, bug→fix, etc.)
- Slug derived from bead title (kebab-case, max 5 words)

### New: Stale branch policy
- Branches older than 7 days with no activity flagged by Witness
- Branches for merged/closed beads pruned by `gt polecat gc`

### New: Screenshot capture for frontend PRs
- Use dev-browser MCP tool to capture before/after screenshots
- Screenshots uploaded and embedded in PR body
- Required when PR touches: `*.tsx`, `*.jsx`, `*.css`, `*.html`, `tailwind.*`
